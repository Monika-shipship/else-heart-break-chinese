# Else Heart.Break() 剧情与翻译参考详解（含剧透）

目的：为翻译与校对提供连贯、统一的剧情脉络与术语锚点。内容基于本仓库 `English/` 文本与文件名线索进行归纳，尽量引用可定位的事件名，帮助在翻译时迅速对齐上下文与人物关系。

注意：本文件包含大量剧透与任务触发信息，建议译者/校对使用，玩家慎读。

## 1. 时间线与章节脉络（按官方章节命名对齐）

- Chapter 1（first day）抵达与入住
  - 家庭餐桌电话引子：主角接到招聘电话，确认将任职 Wellspring Soda 的短期销售，并被要求尽快动身（参考 `_OnThePhone.eng.mtf`）。
  - 船与港口：乘船（船号 340）到达 Dorisburg（参考 `_OnThePhone.eng.mtf`）。
  - 酒店入住：抵达 Hotel Devotchka，与前台 Babcia 办理入住、交押金、出示信用卡、遵守酒店规章（参考 `Babcia_Arrival.eng.mtf`）。
  - 认路与初识城市：旅游办公室领取地图（`TouristOfficeGal_*`），熟悉港口环线、广场等核心区域。
  - 工作身份确立：Wellspring 代表后续出现，强调品牌纪律与岗位规则（`WellspringRepresentant_*`）。

- Chapter 2（after dot）结识圈子与 Lodge 线开启
  - Club Dot 俱乐部：与 Pixie 结识、交谈（`DancingBeauty.eng.mtf`、`Pixie_*`），DJ Petra 常驻，音乐与社交为剧情触发点。
  - 跟随/被误解桥段：Pixie 对“被跟踪”反应强烈，先斥责后缓和，逐渐建立信任（`Pixie_TryFindLodge.eng.mtf` 段落中大量相关对话）。
  - 找 Lodge：Nini（鞋店）作为门卫/联络人；确认后允许通行（`Nini_FoundLodge.eng.mtf`）。
  - 试炼前置：与 Felix、Yulian 等 Lodge 成员交谈，准备试炼（`Felix_*`、`Yulian_HackerTrial1.eng.mtf`）。

- Chapter 3（tests）试炼与入会、基础编程
  - Hacker Trial：由 Pixie 主持或协助，完成若干测试（`*_HackerTrial1/2/3.eng.mtf`）。
  - 取得成员资格：通过后获得 The Lodge 钥匙与多入口权限（含公园大树）（`*_YouAreMemberNow.eng.mtf`，如 `Yulian_YouAreMemberNow.eng.mtf`）。
  - Hank 授课：在 Devotchka 酒店接受 Hank 的 SPRAK 入门指导：接入对象、查看/编译/注入代码、写循环等（`Hank_FirstLecture.eng.mtf`、`Hank_SecondLecture.eng.mtf`、`TeachLoops.eng.mtf`）。

- Chapter 4（factory）潜入工厂与装置操作
  - 工厂任务：学习写循环与控制设备，处理门禁/陷阱/传感等（`Frank_FactoryMission.eng.mtf`、`Frank_GotIntoFactory.eng.mtf`、`_DetectFactoryBreakIn.eng.mtf`）。
  - Wellspring 仓储示范：代表带看控制仓库门的电脑，密码示例（abc123/123abc）等（`WellspringRepresentant_ShowingTheStorage.eng.mtf`）。

- Chapter 5（experiment）网络与“部委”线索
  - 上网与前哨：通过 Slurp() 进入网络世界，Araki 介绍网络前哨与部委的安保（`Araki_AfterGhostVisit.eng.mtf`、`*_ReachedInternet.eng.mtf`）。
  - 风险提示：长时间停留或异常退出的负面效果（`_DieOnInternet.eng.mtf` 等）。

- Chapter 6（final act）对抗与抉择
  - Queen of the Internet：与“网络女王”的对峙与驱逐警告（`Monad_QueenOfTheInternet.eng.mtf`）。
  - 城政/部委调查支线：与 George 等对话牵出“市长/部委”信息公开与对抗（`George_InvestigateMayor.eng.mtf`）。
  - 赌场线收束：Casino Night → Heist → NightIsOver 的后续（`*_GamblingTonight/CasinoHeist/CasinoNightIsOver.eng.mtf`）。
  - 结局走向：受任务完成情况、站队取向与你对系统的“改写深度”影响（本仓库文本未完全覆盖结尾细节，建议以一致译法保留开放性）。

## 2. 角色档案与台词风格

- Sebastian（主角）
  - 人设：初来乍到、好奇心强、从销售员成长为“会编程的人”。
  - 语气：礼貌、探索型，逐步自信。
- Pixie（关键）
  - 人设：外向、自主、对被跟踪零容忍；对主角态度从防备 → 谅解 → 认可。
  - 语气：直率、情绪强、口头禅多，称呼“Seb/Sebbe”。
- Yulian（Lodge 领导）
  - 人设：谨慎、多疑，强调安全与“无部委关联”。
  - 语气：干脆、权威、略带威慑。
- Felix（同伴/背调）
  - 人设：老练、打圆场；常替人“背书”。
  - 语气：松弛、幽默、俚语多。
- Hank（导师）
  - 人设：酒店勤杂，实践派导师；“把东西修好”是其工作理念延伸到代码世界。
  - 语气：朴实耐心、鼓励式讲解。
- Nini（鞋店）
  - 人设：门卫/联络；对熟人友好，做事讲规矩。
  - 语气：随意但不失分寸。
- Babcia（酒店前台）
  - 人设：执规矩、注重流程；押金/信用卡很严格。
  - 语气：正式克制。
- Frank / Araki / George / Ivan / Petra（略）
  - Frank：工厂与赌场线，懂“破限”诀窍。
  - Araki：网络前哨、安保与 Slurp() 指引。
  - George：市政/部委侧线信息人。
  - Ivan：网络事件焦点人物。
  - Petra：DJ，Pixie 的好友，俱乐部氛围关键。

翻译提示：称呼、昵称（如 Seb、Sebbe）、口头禅（“Mhm…/Eh…”）、语气强度（感叹号、脏话）要保持一致。

## 4. 系统与物件的“可改写性”要点

- 修改器（modifier）权限与破限
  - 现象：初期设备有“人工限制”，如“不能连门”。
  - 破限方法：利用“第二个设备修改第一个设备”的思路（“修改器不能修改自己，但能修改别的修改器”）；详见 `Frank_CasinoHeist.eng.mtf` 中的指导对话。
- 常见可控对象
  - 门/锁（Lock/Toggle/Keys API）、灯（PowerTap API）、电梯、气象（Meteorology）、人物数值（Heart/MainFrame）等。
- 代码实践路径
  - 从 Print()/Say() → 读取对象状态 → 控制单个对象 → 设计“对象间联动”。
