# Else Heart.Break() 背景信息汇总

本文整合本仓库“背景信息”文件夹中的资料（SPRAKManual、FullSPRAKCommandReference、WelcomeToDorisburg、ReadBeforePlaying），供翻译与游玩参考。尽量避免剧透，仅给出必要的概览与实用线索。

## 游戏概览
- 核心结构：
  - 冒险探索：3D 点按式城市场景（Dorisburg），对话、跑图、解锁地点与道具。
  - 编程改造：通过“SPRAK”语言读取与改写世界内对象（电脑/门/灯等）的代码与参数，改变规则与行为。
- 设计理念：
  - 鼓励好奇、实验与观察；世界是一个可互动、可修改的系统。
  - 强烈剧透敏感；推荐先自行探索，再查资料。
- 早期目标：
  - 乘坐编号 340 的船抵达 Dorisburg → 办理酒店 Devotchka → 逐步接触“修改器/电脑” → 习得 SPRAK 基础。

## Dorisburg 城市与导航（Welcome to Dorisburg）
- 地图与定位：
  - 旅游办公室可领取地图；背包里有地图时按 `M` 打开。
  - 打开地图时视角会自动朝北；合上地图后不转镜头即可按方向行进。
  - 城市由若干分区构成，黑框分割；双红线为公共交通环线（内圈顺时针，外圈逆时针）。
- 重要地点：
  - 酒店 Devotchka：初期落脚点。
  - 港口与环线交通：快速串联主要街区。
  - 广场/商业区：任务与线索密集处。
- 新手路径建议：
  - 刚下船优先熟悉酒店与城市中轴，了解交通与指路标识。
  - 不必急于攻略，先“认路”和记地点名称（为后续编程检索对象名做准备）。

## 游玩心态与入门建议（Read Before Playing）
- 两条“规则”：
  - 规则 1：自己发现（Find out yourself）。主动搜集信息并记录：人名、物品、房间/门名、电脑名、代码片段等。
  - 规则 2：打破规则（Break the rules）。超越直觉的玩法，用一个对象去影响另一个，联动产生第三种效果。
- 信息策略：
  - 获取：多观察多试验，搞清“如何找到答案”的方法本身。
  - 管理：用笔记/文本/表格保存线索，便于回溯与组合。
- 工具与道具：
  - 便携、可复用的物品价值极高（如能解锁/读取/连接的器材）。
  - 难找的关键物品多在“难进入的房间”里，注意探索与时间节点。
- 元玩法（Meta）：
  - 熟悉 SPRAK 后可进一步研究外部文件结构（建议至少通一遍再深入）。
  - 二周目可尝试不同路线/职业选择/最快获得修改器等挑战。

## SPRAK 常用命令速查（Full SPRAK Command Reference）
（按用途摘选，实际可用指令远多于此——详见原参考文档）
- 输出与交互：
  - `Print(s)` 打印文本；`PrintS(s)` 不换行；`Input()` 获取输入；`Say(s)` 屏幕提示/对话框。
  - `CopyToClipboard(s)` 复制到系统剪贴板。
- 世界与对象：
  - `GetRoom(name)`、`GetPosition(name)` 查询对象所处房间与坐标。
  - `GetPeople()`、`GetThingsInRoom(room)` 遍历人物/物体。
  - `SetWorldPosition(room, x, y)`、`Teleport(x, y)` 移动角色/对象；`Goto(name)` 传送到对象。
- 设备与环境：
  - `TurnOn(name)` / `TurnOff(name)` / `Switch(name)` 控灯；`Lock(code)` / `Toggle(code)` 控门；`SetRain(v)` 控雨量。
  - `SetMhz(v)`、`SetMaxTime(v)` 调整电脑性能/执行时间限制。
- 数据与流程：
  - `Count(arr)`、`Range(a, b)`、`Random()`、`StringContains(s, sub)`。
  - `Sleep(sec)` 延时；`EnableAPI(name)` 启用目标电脑 API；`Connect(name)` / `DisconnectAll()` 网络连接管理。
- 人物与数值：
  - `GetSleepiness(name)`、`SetNumericData(entity, key, val)` 修改数值字段。
- 远程与高级：
  - `RemoteFunctionCall(conn, funcName, args[])` 远程调用。
  - `RestoreCode(name)` 恢复某电脑为默认代码（用于“救砖”）。

## 快速上手编程思路
- 找到可交互的计算机（显示名/房间名是线索），查看其代码，理解输入/输出与 API。
- 从“无害修改”开始：打印、显示提示、记录位置、读取参数。
- 再到“轻度控制”：开关灯/门、改变移动速度、时间加速等。
- 逐步尝试“系统级联动”：通过一个对象的事件触发另一个对象的行为。
